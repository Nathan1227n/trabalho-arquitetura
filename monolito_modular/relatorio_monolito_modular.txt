Relatório de alterações do módulo monolito_modular

Data: 1 de junho de 2026

Resumo do que foi feito:

1. Corrigido o problema de inicialização do Django:
   - Adicionado o arquivo ausente `pagamento/urls.py`.
   - Ajustada a inclusão de URLs para que o projeto carregue corretamente.

2. Finalizado o módulo de notificação:
   - Criado `notificacao/apps.py` para tornar o app Django válido.
   - Criado `notificacao/models.py` e `notificacao/admin.py` como estrutura mínima do app.
   - Criado `notificacao/views.py` com health check.
   - Criado `notificacao/urls.py` e incluído em `core/urls.py`.
   - Criado `notificacao/services.py` com `NotificacaoService.avisar_cozinha(...)`.

3. Melhorado o fluxo de pedidos:
   - `PedidoService` agora usa `NotificacaoService` após pagamento.
   - Adicionado `PedidoService.listar_pedidos()`.
   - Adicionado `PedidoService.cancelar_pedido(pedido_id)`.

4. Atualizadas as views de pedidos:
   - `PedidoAPIView` agora suporta listagem (`GET /pedidos/`) e criação (`POST /pedidos/`).
   - Adicionado `PedidoDetailAPIView` para cancelamento com `DELETE /pedidos/<pedido_id>/`.
   - Ajustadas mensagens e respostas para refletir o fluxo de negócio.

5. Tornado o serviço de pagamento mais completo:
   - `PagamentoService` passou a registrar status dos pagamentos internamente.
   - Adicionado método `consultar_status(pedido_id)`.
   - O delay de 5 segundos foi mantido para atender ao experimento obrigatório.

6. Criado README específico do módulo `monolito_modular`:
   - Documenta como rodar, endpoints, arquitetura e limitações.
   - Esclarece que este README cobre apenas o módulo modular monolítico.

7. Validação final:
   - Executado `./venv/bin/python3 manage.py check` com sucesso.

Observações importantes:
- A arquitetura `monolito_modular` usa um único banco SQLite compartilhado.
- A separação de schema é simbólica via prefixos no `db_table` das tabelas.
- A notificação ainda não usa fila; ela é chamada internamente após o pagamento.
- As outras versões do trabalho (monolito tradicional e microsserviços) não foram alteradas e não fazem parte deste relatório.
